快速上手，如何DIY自己的部落格!!

發表於 2018-07-11 | 更新於: 2019-01-12 | 分類於 Blog | | 閱讀次數:

這個Bolg主要是在Windows系統下用GitHub + Hexo + NexT(5.1.4)來架設自己的部落格，
如果你是Linux系統的話，網路上有不少的文章可以參考，甚至還有官方文件可以看，
所以優化跟調整的部分這邊就不多說了。

以下是為建置的步驟，有興趣的人可以參考一下。

申請GitHub

  • 至GitHub官方網站註冊帳號。
  • 付費方式直接用Free就好了。
  • 驗證Email。

建立Blog專用項目

  • 在GitHub上開一個新的Repositories。
  • 命名為 your-github-username.github.io.git

your-github-username 你的帳號名稱。

安裝 git

  • 至 git-scm 官方網站。
  • Download 2.18.0 for Windows。

安裝 node.js

  • 至 Node.js 官方網站。
  • Download 8.11.4 LTS for Windows (x64)。

這邊有兩種版本，我們直接選"Recommended For Most Users"的版本就好。

安裝 Hexo

當git跟node.js安裝完畢後，就可以透過npm來安裝Hexo。

初始化Hexo

當Hexo安裝完之後，執行下列指令，讓Hexo開始進行部落格環境的初始配置。

<folder> 部落格的資料夾位置。

部落格的開張

當整個初始配置完成之後，會在安裝的目錄下面看到以下的檔案配置。

此時可以執行下列指令，測試部落格是否可以正常運行。

hexo預設的網址為http://locallhost:4000/

接下來只要在網頁上輸入以上網址，噹噹噹，你將會看到Hexo預設的網頁畫面，
到這邊，先恭喜你，已經完成一大半了。

如果在網頁上輸入以上網址卻沒作用的話，請檢查一下是否有本基端的port被佔用了。

如果port真的被佔用或著想要直接指定port的位置，可以參考以下的指令。

s(server): 啟動伺服器
-p(port): 所要指定的port

創建新文章

根據設定的layout模板，將產生出來的文章檔案分別放在

source/_drafts/、source/_posts/、source/下。

Hexo 有三種預設 layout: draft(草稿)、post(公開文章) 和page(頁面)，
如果下指令沒有特別指定layout的話，基本的預設是post。

如果是window系統的話，不用這麼麻煩，可以直接copy官方預設的.md文章，
改個名稱丟到指定的目錄下，一樣可以運作的。

編輯文章

產生出來的文章會是 Markdown(.md) 的格式，
相關的語法可以參考 Markdown語法說明。

市面上有很多Markdown的編輯器，找個用的順手的來編輯文章就好

個人推薦 HackMD

上傳文章

將寫好的文章放到網路空間是基本的常識，所以我們現在要設定Hexo的發佈設定。
還記得最剛開始用github所建立的Repositories嗎?
開啟部落格目錄下的_config.yml檔案，在#Deployment的位置打上你的repo位置。

your-github-username 你的帳號名稱。

設定完之後，我們就可以開始利用以下的指令進行編譯跟發佈了!!

generate: 產生靜態檔案
deploy 發佈靜態檔案到指定位置

Hexo常用指令

Hexo是用指令的方式來進行，看起來好像很複雜，
但其實比較常用的指令就這三個，剩下的其實都可以直接在視窗環境下直接操作。

  • clean: Remove generated files and cache.
  • generate: Generate static files.
  • deploy: Deploy your website.

流程很簡單，基本上就是:
編輯文章 > 產生靜態檔 > 發佈上傳
